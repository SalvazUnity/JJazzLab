JJAZZLAB COMMAND LINE OPTIONS
-J-Dorg.netbeans.modules.autoupdate.level=100   => logging
-J-Dorg.netbeans.modules.autoupdate.ui.actions.AutoupdateSettings.level=100 => logging
-J-Dplugin.manager.check.interval=EVERY_STARTUP  => by default every week
-J-Dplugin.manager.check.new.plugins=true    => check for NEW plugins (not done otherwise, only updates of already installed one)


CHECK FREQUENCY 
A check is done a each startup but nothing done if check interval not reached. By default one week (or 2)!


NBM Signing ==> don't forget to do this for new modules !
The standard Netbeans NBM creation uses the following nbproject/project.properties in each module to sign the module:
keystore=../nbproject/private/JJazzLab-30sept2019.jks
nbm_alias=te-4d8ae524-ca8a-4624-a3f7-39c74a5c08e0
And also in nbproject/private/plaform-private.properties storepass=pwd_to_alias_key_in_keystore
---
How to use build.xml patch see http://wiki.netbeans.org/DevFaqSignNbm:
Define the following properties in Suite's nbproject/project.properties:
- keystore.dname=CN=Jerome Lelasseux, OU=JJazzLab, O=jjazzlab.com, C=FR
- keystore.location=nbproject/private/
- keystore.name=jjazzlab_keystore
- keystore.alias=jjazzlab
Define the following properties in Suite nbproject/private/project.properties:
keystore.password=pwd_to_alias_key_in_keystore
Define the following properties in Suite nbproject/private/platform-private.properties:
keystore.password=pwd_to_alias_key_in_keystore

When running Netbeans Packagas as NBM command the patch will:
- It will create for the Suite a self-signed keystore file ($keystore.location$keystore.name) using the alias (id of the key) and the password defined above, 
- Then for each of the suite module if will update/create :
-- in nbproject/project.properties: the keystore location, and the keystore nbm_alias (the id of the key to search in the store)
-- in nbproject/private/platform-private.properties with storepass=pwd_to_alias_key_in_keystore
- Then it will create and sign all the NBMs

SIGNING WITH JJAZZLAB OWN CERTIFICATE
1.- Convert Windows certificate pfx in jks format (java) and copy it in suite's nbproject/private
1.2 List contents of .pfx:  keytool -list -storetype pkcs12 -keystore JJazzLab-4feb2021.pfx  [one entry with alias like "te-d93210be-c616-4377-a27a-443bd70208d2", hash algorithm eg SHA256] 
1.3 keytool -importkeystore -srckeystore JJazzLab-4feb2021.pfx -srcstoretype pkcs12 -destkeystore JJazzLab-4feb2021.jks -deststoretype JKS 
(garder le mÃªme password pour le jks que pour le pfx)
2. keystore.name=myfile.jfx 
3. keystore.alias=xxx (see above keytool to get the alias name)
Then modify the build.xml patch to NOT generate a keystore since we provide one

=> Use Netbeans Packages as NBM command, it will create and sign all the NBM with our certificate
